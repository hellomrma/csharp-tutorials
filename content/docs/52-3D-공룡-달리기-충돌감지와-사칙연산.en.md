# 52. 3D Dino Run â€“ Collision Detection and Arithmetic System

This tutorial completes the core gameplay of 3D Dino Run. We add sphere-based collision detection (`Physics.OverlapSphere`) to `DinoController` to catch the exact moment the dinosaur passes through a door, and implement the four arithmetic methods in `DinoPositionController` to dynamically adjust the raptor count. We also cover `OnDrawGizmos` to visualize the collision sphere in the editor.

## DinoController â€“ Adding DoorCheck

### Full Code

This is the completed version of the movement code from Tutorial 49, with `DoorCheck` and `OnDrawGizmos` added.

```csharp
using UnityEngine;

public class DinoController : MonoBehaviour
{
    public float zMoveSpeed;
    public float xMoveSpeed;

    public DinoPositionController dinoPositionController;
    public Vector3 sphereCenter;
    public float sphereRadius = 0.5f;

    void Update()
    {
        transform.position += Vector3.forward * Time.deltaTime * zMoveSpeed;

        if (Input.GetKey(KeyCode.LeftArrow))
            transform.Translate(-xMoveSpeed * Time.deltaTime, 0, 0);
        if (Input.GetKey(KeyCode.RightArrow))
            transform.Translate(xMoveSpeed * Time.deltaTime, 0, 0);

        DoorCheck();
    }

    void DoorCheck()
    {
        Collider[] hitColliders = Physics.OverlapSphere(transform.position + sphereCenter, sphereRadius);
        foreach (Collider doors in hitColliders)
        {
            int doorNum = doors.gameObject.GetComponent<SelectDoors>().GetDoorNumber(transform.position.x);
            DoorType doorType = doors.gameObject.GetComponent<SelectDoors>().GetDoorType(transform.position.x);
            doors.gameObject.GetComponent<BoxCollider>().enabled = false;
            dinoPositionController.SetDoorCalc(doorType, doorNum);
        }
    }

    void OnDrawGizmos()
    {
        Gizmos.color = Color.red;
        Gizmos.DrawWireSphere(transform.position + sphereCenter, sphereRadius);
    }
}
```

## Key Concepts â€“ DinoController

### 1. Physics.OverlapSphere â€“ Sphere-Based Collision Detection

```csharp
Collider[] hitColliders = Physics.OverlapSphere(transform.position + sphereCenter, sphereRadius);
```

**`Physics.OverlapSphere`** returns an array of every `Collider` whose bounds overlap the sphere defined by the given center and radius. It is commonly used to detect nearby enemies within a blast radius, or to check whether any objects are within a certain range.

```
Physics.OverlapSphere(center, radius)
                         â†‘        â†‘
              transform.position  sphereRadius
              + sphereCenter
```

**Parameter breakdown:**

| Parameter | Type | Description |
|-----------|------|-------------|
| 1st (center) | `Vector3` | Center of the sphere. `transform.position + sphereCenter` applies an offset from the object's origin |
| 2nd (radius) | `float` | Radius of the sphere. All Colliders within this range are returned |
| 3rd (layerMask) | `int` (optional) | Limit the search to specific layers. Omit to check all layers |

**Return value**: A `Collider[]` array. Returns an empty array (length 0) if nothing is found.

**Why use `sphereCenter`?** If the dinosaur's pivot point is at its feet, you may want to offset the sphere to the torso or slightly in front. Setting `sphereCenter` to something like `(0, 0.5, 1)` in the Inspector moves the sphere forward and up from the object's origin.

### 2. Iterating Over Colliders with foreach

```csharp
foreach (Collider doors in hitColliders)
{
    // ...
}
```

`Physics.OverlapSphere` returns **all** colliders in range as an array. The `foreach` loop processes each one in order.

```csharp
// Equivalent with a for loop
for (int i = 0; i < hitColliders.Length; i++)
{
    Collider doors = hitColliders[i];
    // ...
}
```

In practice only one door is detected at a time, but using `foreach` makes the code safe even if multiple colliders are returned simultaneously.

### 3. GetComponent Chaining â€“ Accessing Components from a Collider

```csharp
int doorNum = doors.gameObject.GetComponent<SelectDoors>().GetDoorNumber(transform.position.x);
DoorType doorType = doors.gameObject.GetComponent<SelectDoors>().GetDoorType(transform.position.x);
doors.gameObject.GetComponent<BoxCollider>().enabled = false;
```

This is a common pattern for accessing other components on the same GameObject that owns the Collider.

```
Collider (doors)
    â””â”€â”€ .gameObject                        â†’ the GameObject this Collider belongs to
         â””â”€â”€ .GetComponent<SelectDoors>()  â†’ the SelectDoors component on the same object
              â””â”€â”€ .GetDoorNumber(...)       â†’ call the method
```

**`doors.gameObject`** retrieves the `GameObject` the `Collider` is attached to. Since `Collider` and `SelectDoors` are on the same object, this is how we reach `SelectDoors`.

For better performance, you can cache the result if you call `GetComponent` on the same object multiple times:

```csharp

DoorType doorType = selectDoors.GetDoorType(transform.position.x);
```

### 4. BoxCollider.enabled = false â€“ Preventing Duplicate Collisions

```csharp
doors.gameObject.GetComponent<BoxCollider>().enabled = false;
```

The door's `BoxCollider` is disabled the moment the dinosaur first touches it. Without this, `DoorCheck` would fire every frame while the dinosaur passes through the door, causing the arithmetic operation to be applied dozens of times in a single pass.

| Situation | Result |
|-----------|--------|
| BoxCollider not disabled | Collision detected every frame during pass-through â†’ operation applied repeatedly |
| `BoxCollider.enabled = false` | Disabled on first contact â†’ operation applied exactly once |

```csharp
// Comparing three approaches
doors.gameObject.GetComponent<BoxCollider>().enabled = false;  // Disable only the Collider (object stays visible)
Destroy(doors.gameObject.GetComponent<BoxCollider>());         // Delete the Collider component
Destroy(doors.gameObject);                                     // Delete the entire object
```

Using `enabled = false` keeps the door visible while stopping further collision detection â€” exactly what we want.

**`OnDrawGizmos`** is a special Unity method that draws editor-only visual aids (gizmos) in the Scene view. **Gizmos are not included in game builds** â€” they are only visible in the Unity editor.

- **`Gizmos.color`**: Sets the color for any gizmos drawn after this line.
- **`Gizmos.DrawWireSphere(center, radius)`**: Draws a wireframe sphere at the given position.

Thanks to this code, selecting the dinosaur object in the Scene view shows a red wireframe sphere, letting you visually confirm the collision range and fine-tune `sphereCenter` and `sphereRadius` without guesswork.

```csharp

```

**`OnDrawGizmosSelected`**: Unlike `OnDrawGizmos`, this only draws when the object is **selected** in the editor. Useful when the scene has many objects and you only want to see gizmos for the currently selected one.

## DinoPositionController â€“ Adding SetDoorCalc

### Full Code

This is the completed version of Tutorial 50's `SetDinoPosition` code, with `SetDoorCalc` and the four arithmetic methods added.

```csharp
using UnityEngine;

public class DinoPositionController : MonoBehaviour
{
    public Transform raptors;
    public float radius = 1f;
    public float ratio = 0.1f;

    public GameObject raptorPrefab;

    void Start()
    {
        SetDinoPosition();
    }

    public void SetDoorCalc(DoorType doorType, int doorNumber)
    {
        if (doorType.Equals(DoorType.Plus)) PlusRaptor(doorNumber);
        else if (doorType.Equals(DoorType.Minus)) MinusRaptor(doorNumber);
        else if (doorType.Equals(DoorType.Times)) TimesRaptor(doorNumber);
        else if (doorType.Equals(DoorType.Division)) DivisionRaptor(doorNumber);
        SetDinoPosition();
    }

    private void SetDinoPosition()
    {
        for (int i = 0; i < raptors.childCount; i++)
        {
            if (i > 8)
            {
                raptors.GetChild(i).gameObject.SetActive(false);
                continue;
            }
            if (raptors.childCount < 10)
            {
                float angleStep = 360f / (raptors.childCount * ratio);
                float angle = i * angleStep;
                float angleRad = angle * Mathf.Deg2Rad;
                float x = Mathf.Cos(angleRad) * radius;
                float z = Mathf.Sin(angleRad) * radius;
                raptors.GetChild(i).localPosition = new Vector3(x, 0f, z);
            }
        }
    }

    private void PlusRaptor(int number)
    {
        for (int i = 0; i < number; i++)
            Instantiate(raptorPrefab, raptors);
    }

    private void MinusRaptor(int number)
    {
        if (number > raptors.childCount) number = raptors.childCount;
        int raptorNum = raptors.childCount;
        for (int i = raptorNum - 1; i >= (raptorNum - number); i--)
            Destroy(raptors.GetChild(i).gameObject);
    }

    private void TimesRaptor(int number)
    {
        int currentCount = raptors.childCount;
        int toAdd = currentCount * number - currentCount;
        for (int i = 0; i < toAdd; i++)
            Instantiate(raptorPrefab, raptors);
    }

    private void DivisionRaptor(int number)
    {
        int currentCount = raptors.childCount;
        int toRemain = currentCount / number;
        if (toRemain < 0) toRemain = 0;
        int toRemove = currentCount - toRemain;
        for (int i = currentCount - 1; i >= (currentCount - toRemove); i--)
            Destroy(raptors.GetChild(i).gameObject);
    }
}
```

## Key Concepts â€“ DinoPositionController

### 6. SetDoorCalc â€“ Branching by Operation Type

```csharp
public void SetDoorCalc(DoorType doorType, int doorNumber)
{
    if (doorType.Equals(DoorType.Plus)) PlusRaptor(doorNumber);
    else if (doorType.Equals(DoorType.Minus)) MinusRaptor(doorNumber);
    else if (doorType.Equals(DoorType.Times)) TimesRaptor(doorNumber);
    else if (doorType.Equals(DoorType.Division)) DivisionRaptor(doorNumber);

}
```

`DinoController` passes the `doorType` and `doorNumber` to this method, which calls the appropriate arithmetic method. After the raptor count changes, `SetDinoPosition()` is called again to recalculate the circular arrangement for the new count.

### 7. PlusRaptor â€“ Adding Raptors with Instantiate

```csharp
private void PlusRaptor(int number)
{
    for (int i = 0; i < number; i++)
        Instantiate(raptorPrefab, raptors);
}
```

**`Instantiate(prefab, parent)`** creates a copy of the prefab and sets the given Transform as its parent.

```csharp
// Instantiate overloads
Instantiate(prefab);                                   // At world origin, no parent
Instantiate(prefab, position, rotation);               // Set position and rotation, no parent
Instantiate(prefab, parent);                           // Set parent, placed at parent's local origin
Instantiate(prefab, position, rotation, parent);       // Set position, rotation, and parent
```

Passing `raptors` as the parent means each newly created raptor becomes a child of `raptors`, immediately increasing `raptors.childCount`. The subsequent `SetDinoPosition()` call then uses the updated count to recalculate the circular layout.

Example: 3 raptors currently, pass through a `+2` door â†’ `PlusRaptor(2)` â†’ Instantiate twice â†’ 5 raptors.

### 8. MinusRaptor â€“ Why Delete from the Back

```csharp
private void MinusRaptor(int number)
{
    if (number > raptors.childCount) number = raptors.childCount;
    int raptorNum = raptors.childCount;
    for (int i = raptorNum - 1; i >= (raptorNum - number); i--)
        Destroy(raptors.GetChild(i).gameObject);
}
```

**Guard clause**: If `number` exceeds `raptors.childCount`, it is clamped to `childCount`. For example, if only 2 raptors remain and the player hits a `-5` door, the count drops to 0 rather than going negative.

**Why delete from the back (decrementing `i` from `raptorNum - 1`)?**

```
Child indices: [0] [1] [2] [3] [4]   (5 raptors)

Deleting from the front:
  Delete [0] â†’ children re-index to [0][1][2][3]
  Delete [0] â†’ children re-index to [0][1][2]
  â†’ Index shifts can cause unintended objects to be deleted

Deleting from the back:
  Delete [4] â†’ [0][1][2][3] (front indices unchanged)
  Delete [3] â†’ [0][1][2]    (front indices unchanged)
  â†’ Safely removes exactly the desired number of raptors
```

```csharp
// Example: delete 2 of 5 raptors
int raptorNum = 5;
// i = 4 â†’ destroy GetChild(4)
// i = 3 â†’ destroy GetChild(3)
// i = 3 >= (5 - 2) = 3, so loop ends
```

`Destroy` executes at the end of the current frame, meaning `childCount` does not decrease immediately during the loop. Iterating from the back ensures index stability throughout.

### 9. TimesRaptor â€“ Multiplying the Raptor Count

```csharp
private void TimesRaptor(int number)
{
    int currentCount = raptors.childCount;
    int toAdd = currentCount * number - currentCount;
    for (int i = 0; i < toAdd; i++)
        Instantiate(raptorPrefab, raptors);
}
```

**Formula**: `raptors to add = current count Ã— multiplier âˆ’ current count`

```
3 raptors currently, pass through Ã—2 door:
Target = 3 Ã— 2 = 6 raptors
To add = 6 âˆ’ 3 = 3 Instantiate calls
```

| Current | Multiplier | Target | To Add |
|---------|-----------|--------|--------|
| 3 | 2 | 6 | 3 |
| 5 | 3 | 15 | 10 |
| 4 | 1 | 4 | 0 (no change) |

**Why save `currentCount` before the loop?** Every call to `Instantiate` increases `raptors.childCount`. If the loop condition used `raptors.childCount` directly, it would keep growing and the loop would never end. Capturing the initial value in a variable fixes the target.

### 10. DivisionRaptor â€“ Integer Division and Edge Cases

```csharp
private void DivisionRaptor(int number)
{
    int currentCount = raptors.childCount;
    int toRemain = currentCount / number;
    if (toRemain < 0) toRemain = 0;
    int toRemove = currentCount - toRemain;
    for (int i = currentCount - 1; i >= (currentCount - toRemove); i--)
        Destroy(raptors.GetChild(i).gameObject);
}
```

**Integer division**: In C#, dividing two `int` values truncates any decimal portion.

```csharp
int result = 5 / 2;  // 2  (not 2.5)
int result = 7 / 3;  // 2  (not 2.33...)
```

```
5 raptors currently, pass through Ã·2 door:
Remaining = 5 / 2 = 2  (integer division)
To remove = 5 âˆ’ 2 = 3 raptors
```

| Current | Divisor | Remaining (int div) | To Remove |
|---------|---------|---------------------|-----------|
| 6 | 2 | 3 | 3 |
| 5 | 2 | 2 | 3 |
| 7 | 3 | 2 | 5 |
| 4 | 4 | 1 | 3 |

**`if (toRemain < 0) toRemain = 0`**: In theory, dividing two positive integers can never produce a negative result, but this guard clause is an example of defensive programming â€” it ensures the value can never go below zero regardless of future changes.

Deletion is performed from the back for the same index-stability reasons explained in `MinusRaptor`.
